How to Set Up Avid–Adobe Premiere Workflow for PAM Integration

By The Streamic Editorial Team Updated March 2025 ⏱ 20 min read 🏗️ Multi-editor / Facility

Many broadcast facilities run Avid Media Composer and Adobe Premiere Pro side by side — news and long-form editorial on Avid, digital content and social delivery on Premiere. When both NLEs need to share media via a Production Asset Management system (PAM), getting the integration right from the start avoids the version conflicts, metadata loss, and media duplication that plague poorly planned mixed-NLE environments.

1

Configure Shared Storage Accessible by Both NLEs

The foundation of any mixed NLE workflow is a storage volume that both Avid and Premiere workstations can read and write simultaneously. Avid NEXIS is the cleanest solution — the NEXIS Client software (which must be installed on every Premiere workstation) presents the NEXIS workspace as a local volume, and Avid's media engine handles concurrent access natively.

If NEXIS is not in your facility, a high-performance SMB share on a NAS (such as a Quantum Xcellis or EditShare) works for most frame rates, though you lose NEXIS's bandwidth reservation guarantees. On a SMB-based workflow, do not have both Avid and Premiere write to the same Avid MediaFiles folder simultaneously — this corrupts the MXF index. Instead, use separate subfolders per NLE.

Recommended folder structure on shared volume:
/SharedMedia/
/Avid_MediaFiles/MXF/1/ ← Avid writes here
/Premiere_Media/ ← Premiere writes here
/PAM_Ingest/ ← PAM ingest watch folder
/PAM_Deliver/ ← Finished deliverables
Tip: Set NTFS or POSIX permissions so the Avid service account and the Adobe Creative Cloud user account both have full read/write access to all subfolders. Permissions mismatches are the most common cause of mysterious "media offline" errors on startup.
2

Install AMA Plug-ins and the Avid Panel for Premiere

On each Avid workstation, ensure the latest AMA (Avid Media Access) plug-ins are installed. These allow Media Composer to link natively to camera original formats without transcoding. Go to Avid's downloads page and install the AMA bundle appropriate for your camera formats (Sony, Canon, Panasonic, RED, etc.).

On each Premiere workstation, install the Avid panel for Adobe Premiere Pro from Avid's website or via the Adobe Exchange marketplace. This panel gives Premiere editors direct access to the PAM's MediaCentral asset browser — they can drag and drop assets from the PAM into a Premiere sequence without leaving the application.

Additionally, install the Avid NEXIS Client on all Premiere workstations (if using NEXIS storage). This is a separate download from the Media Composer installer and must be kept in version sync with the NEXIS server firmware.

Important: Avid NEXIS Client version must match the NEXIS server firmware minor version. A mismatch — even one minor version — prevents the client from mounting the workspace. Always check the NEXIS compatibility matrix before updating either component.
3

Set Up a Common Media Format Both NLEs Can Use

The safest interchange format between Avid and Premiere in a PAM environment is MXF/OP-1a wrapped DNxHD or DNxHR. Both NLEs read and write this format natively without re-wrapping, and PAM systems including Dalet, Grass Valley AMPP, and Ross Mira handle it cleanly.

In Media Composer, set your project's Media Creation settings to output MXF/OP-1a. In Premiere, go to Preferences → Media and ensure Write MXF files using OP-1a is enabled. Both NLEs will then produce files the PAM can ingest, transcode for proxy, and deliver to either editorial system without format conversion.

For facilities requiring ProRes (common in Apple Silicon Premiere deployments), note that Avid Media Composer on macOS can read ProRes but produces MXF-wrapped media natively. For a clean roundtrip, use Avid's media consolidation to convert Premiere's ProRes files to DNxHR before handing work to Avid editors.

Tip: Choose DNxHR SQ (Square/Standard Quality) at your target resolution as your common working codec. It delivers excellent quality at manageable file sizes, plays back without hardware acceleration on most modern workstations, and is universally supported by PAM ingest systems.
4

Map PAM Metadata Fields to Both NLEs

A PAM is only as useful as its metadata — and in a mixed NLE environment, metadata fields must be mapped consistently across both systems to prevent duplication of effort and metadata loss during handoffs.

In Avid Media Composer, custom bin columns can be mapped to PAM metadata fields via the MediaCentral panel configuration. Work with your PAM vendor to define the field mapping. Common fields to map: Programme Title, Story Slug, Shoot Date, Camera Roll, Clip Description, Rights Category.

In Adobe Premiere, the XMP metadata schema is used. The Avid panel for Premiere writes PAM metadata into Premiere's metadata panel as XMP properties. Configure the metadata display in Premiere (Clip → Metadata Display) to show the same core fields your Avid editors see in their bin columns.

Key fields to map in both systems:
Programme Title → Avid: Bin Column "Programme" | XMP: dc:title
Shoot Date → Avid: Date column | XMP: xmp:CreateDate
Camera Roll → Avid: "Tape" field | XMP: xmpDM:tapeName
Rights Category → Avid: Custom column | XMP: xmpRights:Marked
5

Configure the AAF Roundtrip for Project Handoff

When an Avid editor needs to hand a sequence to a Premiere editor (or vice versa), the standard interchange format is AAF (Advanced Authoring Format). Both NLEs support AAF export and import, but the settings matter significantly.

From Avid to Premiere: In Media Composer, select the sequence and go to File → Export → AAF. Choose Embed Media if you want the AAF to be fully self-contained, or Link to existing media if Premiere workstations have access to the same shared storage. The latter is preferred in a PAM environment as it avoids duplicating large media files.

From Premiere to Avid: In Premiere, use File → Export → Final Cut Pro XML and then convert using DaVinci Resolve as an intermediary — Premiere's native AAF export has historically had compatibility issues with Avid's AAF importer. Alternatively, use the AATranslator utility, which provides the most reliable Premiere-to-Avid AAF conversion available.

Note: Nested sequences, dynamic linked items, and Lumetri colour grades do not survive the AAF roundtrip. Flatten all nested sequences and render colour grades before exporting AAF for Avid delivery.
6

Set Up PAM Watch Folders for Automated Ingest and Delivery

With the NLE integration in place, the final step is configuring the PAM's automated watch folder system so that finished sequences from both Avid and Premiere are automatically ingested, transcoded for proxy, and catalogued without manual intervention.

Configure a PAM ingest watch folder on the shared storage — typically the /PAM_Ingest/ directory. Point both Avid's Send to PlayMaker / Send to FluidCart export target and Premiere's Adobe Media Encoder Queue output to this folder.

In your PAM system, set up an automated ingest rule: when a new MXF file appears in the watch folder, ingest it, generate a low-res proxy, apply metadata from the filename or embedded sidecar XMP/XML, and add it to the shared asset library accessible from both NLE panels.

Tip: Use a consistent filename convention that encodes programme name, date, and a unique clip ID — for example: PROGNAME_20250317_CAM01_0042.mxf. PAM systems can parse structured filenames to auto-populate metadata fields at ingest, significantly reducing manual logging work.
